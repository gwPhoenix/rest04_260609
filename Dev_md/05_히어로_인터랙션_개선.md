# 05. 히어로 인터랙션 개선 및 타이포그래피 조정

작업일: 2026-06-09

## 목표
히어로 영역에 시각적 몰입감을 높이는 JS 애니메이션 효과를 추가하고,
지나치게 큰 텍스트를 가독성에 맞게 조정한다.

## 변경 내역

### 1. 파티클 신경망 Canvas 애니메이션 추가

**파일**: `src/components/ParticleCanvas.jsx` (신규)

#### 동작 방식
- 65개의 파티클이 히어로 영역에서 자유롭게 떠다님
- 파티클 간 거리 160px 이하 → 연결선 자동 생성
- 마우스 커서 260px 이내 파티클 → 마우스로 연결선 생성 (인터랙션)
- AI 신경망 시각화 컨셉과 어울리는 효과

#### 기술 구현
```jsx
// ResizeObserver로 캔버스 크기 자동 조정
const ro = new ResizeObserver(resize)
ro.observe(canvas)

// 60프레임마다 팔레트 색상 재읽기 (--brand-400 CSS 변수)
if (++colorFrame % 60 === 0) colorCache = readBrandRgb()

// 마우스 위치는 Hero 섹션에서 추적 후 ref로 전달
// pointer-events: none → 버튼 클릭 방해 없음
```

#### 최적화
| 환경 | 파티클 수 |
|---|---|
| 데스크탑 (768px 이상) | 65개 |
| 모바일 (768px 미만) | 36개 (55%) |

#### 팔레트 연동
- `getComputedStyle(document.documentElement).getPropertyValue('--brand-400')` 로 현재 팔레트 색상 읽기
- 팔레트 전환 시 파티클/선 색상 자동 변경

### 2. Hero 섹션 구조 개선

마우스 트래킹을 위해 Hero 컴포넌트에 이벤트 핸들러 추가:

```jsx
const mouseRef = useRef({ x: -9999, y: -9999 })

const handleMouseMove = useCallback(e => {
  const rect = e.currentTarget.getBoundingClientRect()
  mouseRef.current = { x: e.clientX - rect.left, y: e.clientY - rect.top }
}, [])
```

레이어 순서:
1. `HeroBg` — 그라디언트 배경 (z: auto)
2. `ParticleCanvas` — 파티클 애니메이션 (pointer-events: none)
3. 텍스트 콘텐츠 — (z-10)
4. 컨트롤 버튼 — (z-10)

---

### 3. 히어로 텍스트 크기 축소 (가독성 개선)

**문제**: `xl:text-[8rem]` (128px) — 화면을 가득 채워 가독성 저하

| 브레이크포인트 | 이전 | 이후 |
|---|---|---|
| 기본 (모바일) | `text-5xl` = 48px | `text-4xl` = 36px |
| md (768px+) | `text-7xl` = 72px | `text-5xl` = 48px |
| lg (1024px+) | `text-8xl` = 96px | `text-6xl` = 60px |
| xl (1280px+) | `[8rem]` = 128px | `text-7xl` = 72px |

`leading-tight` → `leading-snug` 으로 변경해 줄간격 여유 추가
